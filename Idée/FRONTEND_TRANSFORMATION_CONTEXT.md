# HAFROSE — Frontend Luxury Transformation Context

Version : 1.0
Projet : HAFROSE
Statut : Document de référence
Priorité : Critique

---

# 1. Objectif

Le frontend actuel est fonctionnel mais ne reflète pas l'image d'une véritable Maison de Luxe.

L'objectif de cette transformation est de faire évoluer HAFROSE vers une expérience digitale premium, inspirée des grandes maisons telles que :

- Dior
- Louis Vuitton
- Cartier
- Hermès
- Chanel

Le résultat final doit transmettre :

- Élégance
- Raffinement
- Modernité
- Simplicité
- Exclusivité
- Confiance
- Désir d'achat

Chaque écran doit donner au client l'impression de visiter une véritable boutique de luxe.

---

# 2. Philosophie de Design

Toutes les décisions de design doivent respecter les principes suivants.

## Minimalisme

Éviter toute surcharge visuelle.

Chaque élément doit avoir une utilité.

Le vide est considéré comme un élément de design.

---

## Élégance

Le design doit être léger.

Les animations doivent être discrètes.

Les transitions doivent être fluides.

Les composants doivent être raffinés.

---

## Cohérence

Toutes les pages doivent partager exactement la même identité visuelle.

Aucune page ne doit avoir son propre style.

L'utilisateur ne doit jamais avoir l'impression de changer de site.

---

## Premium

Chaque détail doit inspirer confiance.

Même un bouton ou un champ de formulaire doit paraître haut de gamme.

---

# 3. Objectifs UX

Le visiteur doit ressentir :

✔ confiance

✔ qualité

✔ exclusivité

✔ simplicité

✔ fluidité

✔ confort

✔ envie d'explorer

✔ envie d'acheter

✔ envie de revenir

---

# 4. Vision

HAFROSE n'est pas un simple e-commerce.

HAFROSE est une Maison de Haute Maroquinerie.

Le site doit raconter une histoire.

Les produits ne sont pas simplement vendus.

Ils sont mis en valeur.

---

# 5. Inspiration

Les interfaces doivent s'inspirer de :

- Dior
- Cartier
- Louis Vuitton
- Hermès
- Saint Laurent

Sans copier leur design.

L'objectif est de retrouver leur niveau de qualité.

---

# 6. Direction Artistique

Le style général doit être :

- Luxueux
- Féminin
- Moderne
- Raffiné
- Épuré
- Haut de gamme

Éviter :

- les couleurs agressives
- les effets flashy
- les animations excessives
- les gradients très marqués
- les composants trop techniques

---

# 7. Palette de Couleurs

Le thème officiel est :

Rose poudré

Rose Gold

Blanc cassé

Beige

Or discret

Anthracite

Le jaune actuellement présent dans le projet doit disparaître.

Toutes les couleurs devront être centralisées dans le Design System.

---

# 8. Typographie

Les titres doivent transmettre l'élégance.

Les textes doivent rester très lisibles.

Les tailles de police devront être harmonisées sur l'ensemble du projet.

Aucune page ne doit utiliser un style différent.

---

# 9. Composants

Tous les composants devront être réutilisables.

Par exemple :

- Navbar
- Footer
- Buttons
- Cards
- Inputs
- Search
- Filters
- Hero
- Product Cards
- Section Title
- Newsletter
- Modal
- Drawer

Aucun composant ne doit être dupliqué.

---

# 10. Animations

Les animations doivent être :

- lentes
- fluides
- élégantes

Éviter :

- les animations brusques
- les rotations inutiles
- les rebonds

Les animations doivent améliorer l'expérience utilisateur sans distraire.

---

# 11. Responsive

Le site doit être pensé Mobile First.

Tous les composants doivent être parfaitement adaptés :

- Mobile
- Tablette
- Desktop

Aucune exception.

---

# 12. Performance

Chaque amélioration graphique doit préserver :

- la rapidité
- le SEO
- l'accessibilité
- les Core Web Vitals

La qualité visuelle ne doit jamais dégrader les performances.

---

# 13. Cohérence

Toutes les pages doivent partager :

- la même Navbar
- le même Footer
- les mêmes boutons
- les mêmes cartes
- les mêmes animations
- les mêmes espacements
- les mêmes couleurs
- les mêmes icônes
- les mêmes typographies

Le Design System est la seule source de vérité.

---

# 14. Méthodologie

La transformation sera réalisée progressivement.

Une seule phase sera traitée à la fois.

Aucune phase ne commencera tant que la précédente n'est pas validée.

Chaque phase devra :

- analyser le code existant
- identifier les composants concernés
- proposer un plan d'action
- implémenter les modifications
- vérifier qu'aucune régression n'est introduite

---

# 15. Règles de Développement

Avant toute modification, l'IA doit :

- analyser les composants existants
- réutiliser les composants lorsque possible
- éviter la duplication de code
- respecter l'architecture actuelle
- conserver les fonctionnalités existantes

Interdictions :

- casser une fonctionnalité
- modifier inutilement le backend
- créer plusieurs styles pour un même composant
- introduire des dépendances inutiles

---

# 16. Définition de la réussite

La transformation sera considérée comme réussie lorsque :

- toutes les pages auront une identité visuelle commune
- le site donnera immédiatement une impression de luxe
- la navigation sera fluide
- l'expérience utilisateur sera cohérente
- les performances resteront excellentes
- le client pourra comparer HAFROSE à une véritable boutique de luxe

---

# 17. Principe Fondamental

Chaque modification doit répondre à cette question :

"Cette amélioration rend-elle HAFROSE plus premium, plus élégante et plus cohérente ?"

Si la réponse est non, la modification ne doit pas être réalisée.
